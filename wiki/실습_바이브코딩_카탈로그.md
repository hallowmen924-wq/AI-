# 실습: 바이브코딩 실습 카탈로그

## 한눈에 보기

`바이브코딩실습/` 은 각 노트북 실습이 끝난 뒤 **클로드코드로 직접 만들어 보는** 28개 실습 모음입니다.
폴더마다 `프롬프트.txt`(수강생에게 보여주는 4단계 프롬프트)와 `app.py`(강사용 완성 예시)가 들어 있으며, 대부분 `streamlit run app.py` 하나로 브라우저에서 동작합니다.

## 핵심 개념

### 프롬프트.txt 4단계 형식 (고정 구조)

```
STEP 0. [탐색]  → 대응 노트북을 읽고 이해만. "아직 아무것도 만들지 마."
STEP 1. [계획]  → 만들 것의 계획만. "코드는 아직 만들지 마."
STEP 2. [구현]  → 브라우저에서 바로 쓸 수 있는 대시보드로 구현
STEP 3. [검증]  → 실행 확인 + (사람 검증: ...) + 핵심 메시지: "..."
```

**설계 의도**: 곧바로 코드를 요구하지 않고 **탐색 → 계획 → 구현 → 검증** 순서를 강제합니다. 그리고 각 실습은 "형식 검증은 AI가, 사실 확인은 사람이 한다" 같은 **핵심 메시지 한 줄**로 끝납니다.

### 모든 앱이 공유하는 코드 규약

| 규약 | 이유 |
|---|---|
| **단일 파일 완결** — `app.py` 하나에 도구·State·그래프·UI 전부 | 수강생이 위에서 아래로 읽음. 모듈 분리 금지 |
| **API 키 하드코딩 금지** — `os.environ.get(...) or st.text_input(..., type="password")` | 보안 원칙 |
| **주석은 한국어**, 상단에 `# 실행: streamlit run app.py` + 대응 노트북 번호 | 추적성 |
| **Mock 데이터는 `(Mock)` 표기** — `FAKE_BALANCE` 등 | 시연 중 "실제 데이터냐" 질문이 반드시 나옴 |
| **실행 경로를 화면에 표시** | 결과뿐 아니라 "어떤 노드/도구를 거쳤는지" 보여주기 |
| **노트북과 동일한 API 사용** | 예: Day03_03은 반드시 `tools_condition`+`ToolNode`. `create_react_agent`로 바꾸면 실습 목적이 깨짐 |

## 핵심 코드 — 표준 앱 구조

`Day03_03_계좌조회챗봇/app.py` 가 모든 규약을 잘 보여주는 표본입니다.

```python
# 계좌 조회 챗봇 대시보드 (조건부 Edge + ToolNode)
# Day03_03 노트북 방식: 챗봇 노드 → tools_condition(조건부 엣지) → ToolNode 또는 END
# 실행: streamlit run app.py

import os
import streamlit as st
from langchain_core.tools import tool
from langgraph.prebuilt import ToolNode, tools_condition

st.set_page_config(page_title="계좌 조회 챗봇", page_icon="💳")
st.caption("Day03_03 조건부 Edge + ToolNode — 상황에 따라 다른 창구로 안내한다")

# ① 키: 환경변수 우선, 없으면 사이드바 입력 (하드코딩 금지)
with st.sidebar:
    api_key = os.environ.get("OPENAI_API_KEY") or st.text_input("OpenAI API 키", type="password")

# ② Mock 데이터 명시
FAKE_BALANCE = 2_345_670
FAKE_HISTORY = ["06-28 스타벅스 -5,600원", "06-29 급여입금 +3,200,000원"]

@tool
def get_balance() -> str:
    """고객의 현재 계좌 잔액을 조회한다."""
    return f"현재 잔액은 {FAKE_BALANCE:,}원입니다."

# ③ 노트북과 동일한 API — tools_condition + ToolNode
def make_graph(llm):
    llm_with_tools = llm.bind_tools([get_balance, get_history])
    builder = StateGraph(ChatState)
    builder.add_node("chatbot", lambda s: {"messages": [llm_with_tools.invoke(s["messages"])]})
    builder.add_node("tools", ToolNode([get_balance, get_history]))
    builder.add_edge(START, "chatbot")
    builder.add_conditional_edges("chatbot", tools_condition)
    builder.add_edge("tools", "chatbot")
    return builder.compile()

# ④ 실행 경로 표시
used_tools = [m for m in result["messages"] if m.type == "tool"]
if used_tools:
    st.info("🧭 경로: chatbot → **tools(도구 창구)** → chatbot → END")
else:
    st.info("🧭 경로: chatbot → END (도구 없이 바로 답변)")

# ⑤ 사람 검증 안내
st.caption(f"🔎 사람 검증: 잔액이 가짜 데이터({FAKE_BALANCE:,}원)와 일치하는지 확인하세요.")
```

## 새 실습을 추가할 때

1. `바이브코딩실습/DayNN_MM_이름/` 폴더 생성

3. `app.py` — 단일 파일, 한국어 주석, 상단에 실행 명령 + 대응 노트북 번호
4. `바이브코딩실습/README.md` 표에 행 추가
5. 이 위키 카탈로그 표에도 행 추가

## 주의사항

- **`app.py` 들에는 키가 하드코딩되어 있지 않습니다.** 이 규약이 잘 지켜져 있습니다. 문제는 **노트북 쪽**입니다 → [[오류_환경설정_트러블슈팅]]
- **모듈을 분리하지 마세요.** 교육 목적상 단일 파일이 원칙입니다. (예외: `Day2/개인실습/검색품질실험실`이 `rag_core.py`를 분리했는데, 테스트를 붙이기 위해서입니다.)
- **앱을 "더 나은 API"로 리팩터링하지 마세요.** 대응 노트북에서 배운 API를 그대로 써야 실습이 연결됩니다.
- **⭕ 표시가 없는 앱은 OpenAI 키가 필요합니다.** 키 없이 시연해야 한다면 ⭕ 앱을 고르세요.
- **Day04_03만 uvicorn**입니다. streamlit으로 실행하면 동작하지 않습니다.
- **Streamlit 앱은 코드가 바뀌면 전체를 다시 실행합니다.** 벡터DB 생성처럼 무거운 작업은 `@st.cache_resource` 로 감싸야 매번 재생성되지 않습니다.
