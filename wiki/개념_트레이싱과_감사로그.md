# 개념: 트레이싱과 감사 로그

## 한눈에 보기

비행기의 블랙박스처럼, AI 서비스도 "누가, 언제, 무엇을 물었고, AI가 어떤 근거로 어떻게 답했는지"를 기록해야 합니다. 금융권에서는 **감사 대응 요건**이기도 합니다.
방법은 3층 — 환경변수만 설정하면 자동 수집되는 **LangSmith/Langfuse**, 토큰·지연을 잡는 **커스텀 콜백**, 그리고 기계가 검색·집계할 수 있는 **JSON 감사 로그**.

## 핵심 개념

| 층 | 도구 | 기록되는 것 |
|---|---|---|
| **플랫폼 트레이싱** | LangSmith(SaaS) / Langfuse(self-host 가능) | 프롬프트·모델·토큰·지연·계층 구조 전체 |
| **커스텀 콜백** | `BaseCallbackHandler` | LLM 시작/종료 시점, 토큰 사용량, 소요시간 |
| **감사 로그** | `logging` + JSON | correlation_id, user_id, timestamp, action, detail |

| 용어 | 뜻 |
|---|---|
| **Trace** | 요청 하나의 전체 처리 과정 |
| **Span** | Trace 안의 개별 단계 (검색 / LLM 호출) |
| **correlation_id** | 요청 하나를 끝까지 따라가는 추적 번호 |
| **Score** | Trace에 붙이는 품질 점수 (자동 평가·사용자 피드백) |
| **`@observe()`** | Langfuse 데코레이터. 함수 전체를 하나의 Trace로 감쌈 |

**LangSmith vs Langfuse — 내부망 관점**

| | LangSmith | Langfuse |
|---|---|---|
| 형태 | 외부 SaaS (smith.langchain.com) | **self-host 가능** (사내 Docker) |
| 내부망 | 데이터가 외부로 나감 → 보안 검토 필요 | 사내에 두면 데이터가 나가지 않음 |
| 교안 권장 | 강의·실습 | **사내 표준 관측 도구** |

실전프로젝트 치환표도 "LangSmith(외부 SaaS) → Langfuse self-host(사내 Docker)"를 명시합니다.

**금융 감사 로그 필수 항목**: 요청 시각, 사용자 ID, 질문 내용(마스킹), 검색된 근거, 최종 답변(마스킹), 사용 모델, 토큰/비용, 처리 경로.

## 전체 동작 흐름

```
[요청]
  ├─ audit_log(user_id, action, detail)  → correlation_id 발급, JSON 한 줄 기록
  ├─ retrieve(question)                   → Langfuse @observe가 span으로 기록
  └─ chain.invoke(..., config={"callbacks": [UsageLogger(), langfuse_handler]})
                                          → 토큰·지연 로컬 출력 + 플랫폼 자동 전송
[응답]
  └─ score_current_trace(name="context_found", value=1.0)   → 품질 점수 축적
```

## 핵심 코드

### 1. LangSmith — 환경변수만으로 자동 수집

```python
LANGSMITH_KEY = os.environ.get("LANGSMITH_API_KEY", "")
if not LANGSMITH_KEY:
    os.environ["LANGSMITH_TRACING"] = "false"
    print("LangSmith 키가 없어 트레이싱은 껐습니다.")
else:
    os.environ["LANGSMITH_TRACING"] = "true"
    os.environ["LANGSMITH_PROJECT"] = "KB-AI-Agent"
```

이후 LangChain의 **모든 LLM/체인 호출이 자동으로** 대시보드에 기록됩니다. 코드 수정은 없습니다.

### 2. 커스텀 콜백 핸들러 ⭐

```python
from langchain_core.callbacks import BaseCallbackHandler
import time

class UsageLogger(BaseCallbackHandler):
    def on_llm_start(self, serialized, prompts, **kwargs):
        self.start = time.time()

    def on_llm_end(self, response, **kwargs):
        elapsed = time.time() - self.start
        usage = (response.llm_output or {}).get("token_usage", {})
        print(f"[USAGE] 총토큰={usage.get('total_tokens')} 소요시간={elapsed:.2f}s")

resp = llm.invoke("정기적금과 정기예금의 차이는?", config={"callbacks": [UsageLogger()]})
```

### 3. 구조화된 JSON 감사 로그 ⭐⭐ (재사용 가치 최상)

운영 로그는 사람이 읽는 문장이 아니라 **기계가 검색·집계할 수 있는 JSON**이어야 합니다.

```python
import logging, json, uuid
from datetime import datetime

logger = logging.getLogger("kb-audit")
logger.setLevel(logging.INFO)
if not logger.handlers:
    logger.addHandler(logging.StreamHandler())

def audit_log(user_id, action, detail):
    record = {
        "correlation_id": str(uuid.uuid4()),      # 요청 추적 번호
        "user_id": user_id,
        "timestamp": datetime.now().isoformat(),
        "action": action,                          # chat_query / rag_retrieve / tool_call
        "detail": detail,
    }
    logger.info(json.dumps(record, ensure_ascii=False))    # 한글이 깨지지 않게
    return record["correlation_id"]

cid = audit_log(user_id="kb-teller-021", action="chat_query", detail="대출 한도 문의")
```

실전프로젝트용 파일 기반 버전(`common/audit.py`) — 데모에서 이 파일을 열어 보여주면 "감사 대응이 되네"라는 반응이 나옵니다:

```python
import json, datetime, pathlib
LOG = pathlib.Path("./logs/audit.jsonl")

def log(**kw):
    LOG.parent.mkdir(exist_ok=True)
    kw["ts"] = datetime.datetime.now().isoformat(timespec="seconds")
    with LOG.open("a", encoding="utf-8") as f:
        f.write(json.dumps(kw, ensure_ascii=False) + "\n")

# 쓰는 법 — 질문·답변은 반드시 마스킹해서 기록
log(user="김대리", question=mask(q), sources=[...], answer=mask(a), path=["classify", "retrieve"])
```

### 4. Langfuse — CallbackHandler (체인 자동 기록)

```python
from langfuse import observe, get_client
from langfuse.langchain import CallbackHandler

langfuse = get_client()
callbacks = [CallbackHandler()]

result = chain.invoke({"question": q, "context": ctx}, config={"callbacks": callbacks})
# ← 이 한 줄로 프롬프트·모델·토큰·지연이 전부 기록됩니다
```

### 5. Langfuse — `@observe`로 함수 단위 계층 기록 ⭐

콜백은 "체인 안"만 기록합니다. 검색까지 포함한 함수 전체를 기록하려면 데코레이터를 씁니다.

```python
@observe()
def rag_answer(user_id, session_id, question):
    langfuse.update_current_trace(
        user_id=user_id,                       # Users 화면에서 고객별 조회
        session_id=session_id,                 # Sessions 화면에서 대화 흐름 조회
        tags=["rag", "kb-loan"],               # 태그로 필터링
    )
    context = retrieve(question)
    if not context:
        answer = "근거 문서를 찾지 못했습니다. 상담원 연결을 도와드릴까요?"
    else:
        answer = chain.invoke({"question": question, "context": "\n".join(context)},
                              config={"callbacks": callbacks}).content
    langfuse.score_current_trace(name="context_found", value=1.0 if context else 0.0)
    return answer
```

계층 구조(함수 → 체인 → LLM)로 표시되어 어느 단계가 느린지 바로 보입니다.

### 6. Score — 품질을 숫자로 축적

```python
@observe()
def rag_answer_with_feedback(user_id, session_id, question, thumbs_up):
    answer = rag_answer(user_id, session_id, question)
    langfuse.score_current_trace(name="user_feedback", value=1.0 if thumbs_up else 0.0)
    return answer

langfuse.flush()      # ← 프로세스 종료 전 반드시 전송
```

| 점수 이름 | 출처 |
|---|---|
| `context_found` | 자동 (근거 문서 확보 여부) |
| `user_feedback` | 사용자 👍/👎 |
| `faithfulness` | 자동 평가 → [[개념_Agent_평가]] |

## 주의사항

- 🚨 **`Day05_03`·`Day05_08` 노트북에 실제 OpenAI·LangSmith·Langfuse 키가 하드코딩되어 있습니다.** Langfuse는 **secret key까지** 노출되어 있습니다 → [[오류_환경설정_트러블슈팅]]
- 🚨 **로그에 원문 개인정보를 남기지 마세요.** 질문·답변을 그대로 기록하면 로그 자체가 개인정보 저장소가 됩니다. `mask()` 를 통과시킨 값만 기록하세요 → [[개념_PII보호와_거버넌스]]
- 🚨 **LangSmith는 외부 SaaS입니다.** 프롬프트와 응답 전문이 외부로 전송됩니다. 내부망·규제 환경에서는 Langfuse self-host를 쓰세요.
- **`user_id`·`session_id`를 감사 로그와 트레이싱에서 같은 값으로 맞추세요.** 이게 어긋나면 사고 조사 때 두 기록을 연결할 수 없습니다. 교안이 명시적으로 강조하는 지점입니다.
- **`langfuse.flush()` 를 빠뜨리면 마지막 Trace가 전송되지 않습니다.** 짧게 실행되는 스크립트·배치에서 특히 주의하세요.
- **`Day05_08` 노트북에 `LANGFUSE_BASE_URL`(cloud)과 `LANGFUSE_HOST`(localhost)가 둘 다 정의되어 있고 실제로는 `HOST`만 사용**합니다. 로컬에 Langfuse가 떠 있지 않으면 전송이 실패하니 주소를 확인하세요.
- **콜백의 `token_usage`는 모델·API에 따라 비어 있을 수 있습니다.** 스트리밍 호출에서는 특히 그렇습니다. `response.usage_metadata` 쪽을 함께 확인하세요.
- **로그 보관 기간과 접근 권한을 정하세요.** 금융권은 보존 기간 규정이 있고, 로그 조회 자체도 통제 대상입니다.
